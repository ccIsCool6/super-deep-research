# 解释性综合：从“读过资料”到“真正讲明白”

本文件是 Deep Research 的核心语义门。它解决的不是“资料里写了什么”，而是研究者在读完资料后应该怎样形成自己的理解，并把足以检查的理由写进成品。

这里要求公开的是**可审计的推理产物**：证据、解释桥梁、假设、替代解释、边界和含义。不要输出私有逐步思维草稿、内部 token 级联想、搜索流水账或未整理的自言自语。读者需要的是能够评价的理由，而不是模型的原始思维记录。

## 一、复述、总结与综合的区别

### 复述

把来源换一种说法：“A 报告认为……B 论文指出……C 专家表示……”。它可能准确，但尚未形成新的理解。

### 总结

压缩来源内容，指出共同主题和主要结论。它减少了信息量，却仍可能没有解释来源为什么一致、为什么冲突、哪些证据更有决定性。

### 解释性综合

围绕用户的核心问题重新组织材料，建立能够同时解释事实、差异和异常的模型。综合至少回答：

- 多个来源共同揭示了什么关系，而不只是各自说了什么；
- 哪些观察是事实，哪些只是作者解释；
- 从证据到结论还需要哪些前提和推断；
- 为什么某种解释比竞争解释更强，或为什么目前不能裁决；
- 结论在哪些条件下成立、变弱或翻转；
- 这个模型会怎样改变读者对新案例的判断。

## 二、写作前的五步二次加工

### 1. 去掉来源顺序

不要按“先读到什么”写作。把材料拆成主张、证据、方法、范围、冲突和未知，再按问题的因果与知识依赖重新组织。

若文章结构与搜索顺序或来源目录相同，通常尚未完成综合。

### 2. 建立最小解释模型

用少量关键变量描述对象如何运作：输入是什么，中间机制是什么，输出是什么，哪些约束和反馈会改变结果。

模型不是追求复杂，而是能够解释：

- 最常见现象；
- 一个重要异常或失败案例；
- 不同来源为何会得出不同结论；
- 条件变化后结果为何变化。

如果一个“模型”只能换词复述结论，不能产生新的预测或边界判断，它还不是解释。

### 3. 写出证据桥梁

对每个决定性判断，内部先写完整：

```text
主张：我们目前判断什么？
证据：来源实际观察、测量、规定或论证了什么？
解释桥梁：为什么这些证据能支持该主张，而不是另一种主张？
必要前提：这一步依赖哪些未被直接证明的假设？
竞争解释：同样证据还能怎样解释？
裁决：为什么保留、降低或拒绝某种解释？
边界：在哪些范围和条件下成立？
含义：这会改变读者的哪个判断？
```

这些关系必须以自然方式进入正文。可以集中成“如何得到这个判断”小节，也可以融入连续段落；不得机械给每段贴八个标签。

### 4. 用异常检查模型

主动寻找最难解释的材料，而不是只吸收支持性材料：反例、负面结果、时间变化、版本差异、样本差异和失败案例。

处理异常有四种诚实结果：

1. 异常来自定义或测量不同；
2. 异常揭示了适用边界；
3. 异常迫使模型增加新变量；
4. 现有证据无法裁决，保留竞争解释。

不得把反例藏进脚注，或用“也存在争议”一句话跳过。

### 5. 从模型生成文章

文章的每一节都应推动中心问题：建立必要概念、解释关键机制、处理证据冲突、收紧边界，或推出含义。章节不是资料分类箱。

判断顺序是否真实承载逻辑：若章节可以随机交换而几乎不影响理解，说明它们只是并列材料，应重新设计结构。

## 三、核心解释单元

对核心概念、机制或判断，按需要覆盖以下六项：

1. **对象与边界**：正在解释什么，又明确不包括什么；
2. **关系与机制**：关键变量如何连接，为什么产生结果；
3. **证据与桥梁**：证据支持哪一步，推断如何完成；
4. **替代解释与裁决**：最强竞争解释是什么，证据如何区分；
5. **案例与边界**：具体情境如何运行，何时失效；
6. **含义与迁移**：读者以后面对新问题应观察哪些变量。

不是每一段都要机械复制六项，但任何决定理解的核心章节都不能系统性缺少机制、证据桥梁和边界。篇幅由把这些关系讲清所需的空间自然产生。

## 四、把“自己的理解”写进报告

研究者可以且应该给出综合判断，但必须让读者分辨它的来源：

- **来源事实**：“数据/文本直接显示……”；
- **来源观点**：“作者将这一现象解释为……”；
- **研究综合**：“把这些结果放在一起，更合理的解释是……理由是……”；
- **有限推断**：“现有材料提示……但仍依赖……假设”；
- **建议**：“如果目标和约束是……那么更可取的是……”；
- **未知**：“现有证据不能判断……需要……才能验证”。

“自己的理解”不等于脱离证据发表随想。它是把多个来源连接、比较、解释和条件化之后形成的判断；越是原创的综合，越要公开推断依据和可翻转条件。

## 五、连续人类文章标准

- 连续推理优先使用段落。每段完成一个主要动作：提出问题、解释机制、展示证据、处理反例、完成转折或推出含义。
- 句子之间要能回答“为什么下一句出现在这里”。只因主题相同而并列的句子尚未形成文章。
- 段落之间要有真实依赖：下一段解决上一段留下的问题、限定其结论、提供证据或推进到下一层。
- 小标题标记思路转折，不是把关键词变成目录。
- 列表只承载平行项目或步骤。若列表项之间有因果、优先级或冲突，应改为段落、流程或比较分析。
- 表格之后必须解释决定性差异；图之后必须说明读者应该看见什么以及它如何改变判断。
- 允许先给方向性结论或全局地图，但完整判断必须沿证据和解释逐步建立。不要为了“认知引导”故意卖关子，也不要把结论提前到读者尚无法理解的位置。

## 六、六项反复述测试

交付前对每个核心章节执行：

### 1. 来源替换测试

如果把来源名称全部替换成“某报告”，这节是否仍有一个清楚、可评价的解释模型？若只剩资料串联，返工。

### 2. 为什么测试

对核心结论连续追问“为什么”和“凭什么”。若下一层只能重复结论或引用权威，缺少解释桥梁，返工。

### 3. 竞争解释测试

能否准确说出最强替代解释，以及什么证据会让当前判断改变？不能则说明研究者只收集了支持材料。

### 4. 异常解释测试

模型能否解释一个失败案例、反例或范围差异？若不能，要么修改模型，要么收窄结论。

### 5. 新案例迁移测试

读者能否根据本节模型判断一个正文未出现的新情境，并说出需要观察的变量？不能则仍停留在记忆结论。

### 6. 连贯性测试

随机交换段落或章节后是否几乎不影响阅读？若是，说明结构没有承载推理依赖，返工重排。

## 七、失败模式

以下任一情况系统性出现，状态为 `REPAIR`：

- 逐篇或逐作者摘要，没有跨来源综合；
- 每一节只有“是什么”，没有“为什么”和“何时不成立”；
- 引用很多，但没有解释引用支持了哪一步；
- 用“因此”“可见”跳过真正的推断桥梁；
- 把常识性改写叫作“独立见解”；
- 只写最终结论，省略读者理解结论所需的关系；
- 把内部草稿、搜索过程或冗长自言自语当成深度；
- bullet、表格和 Callout 占据主体，散文无法独立形成完整论证；
- 各章节像独立知识卡片，缺少共同中心和递进关系；
- 通过重复、外围历史或同义改写增加篇幅。

## 八、审计记录

中度以上在内部为核心章节保留简短记录：

```markdown
- 章节与中心问题的关系：
- 本章解释模型：
- 决定性证据与解释桥梁：
- 最强替代解释及裁决：
- 适用边界和翻转条件：
- 案例/反例承担的功能：
- 读者读完新增的迁移能力：
- 六项反复述测试：PASS / REPAIR
```

最终报告必须呈现上述内容中影响读者理解和判断的部分，但内部审计表本身默认不交付。用户要求过程审计时，可整理为附录；仍不输出私有逐步思维草稿。
