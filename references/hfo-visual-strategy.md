# 视觉表达策略

视觉的价值不在“更高级”，而在用更低成本揭示文字难以同时呈现的关系。先判断视觉是否必要，再选择最小可用媒介。

## 导航

- 何时不使用视觉、何时值得使用
- 最小可用媒介与每张视觉的契约
- 默认升级路径

## 何时不使用视觉

以下内容通常用文字、列表或表格更快：

- 一个事实、结论或定义；
- 一到两个简单步骤；
- 没有结构关系的短枚举；
- 只需展示少量精确值的内容；
- 视觉不会改变理解、判断或行动的装饰性内容。

不要因为文章较长就强制加图，也不要把同一信息同时画成流程图、卡片和表格。

## 何时值得使用

视觉通常适合以下关系：

- 三个以上对象之间的连接、层级或依赖；
- 有分支、并行、循环或状态变化的过程；
- 多对象在相同维度上的重复比较；
- 数值的趋势、分布、相关性、组成或流量；
- 空间位置、时间先后或资源占用；
- 调整输入会显著改变结果的系统；
- 仅靠文字需要读者在脑中反复拼装的整体模型。

## 选择最小可用媒介

### 文档内语义锚点

在创建独立图片前，先判断 Markdown 本身是否已经足够：

- 用小标题表示问题与思路转折；
- 用克制的粗体标出段内关键词或局部判断；
- 用表格比较至少三项对象的重复字段；
- standard Markdown 默认用普通引用块和加粗语义前缀承载旁路信息；只有用户明确选择 Obsidian 配置时，才使用 `[!summary]`、`[!example]`、`[!warning]`、`[!question]`、`[!tip]` 和折叠 `[!note]-`。

Callout 不是“图片配额”的廉价替代。核心论证留在正文，折叠块不隐藏决定性限定，同类 Callout 保持同一意义；离开 Obsidian 后退化为普通引用块也必须可理解。详细规则见 `research-writing-standard.md` 与 `research-post-scan.md`。

### Mermaid

用于标签和连线已经足以解释的静态结构，例如简单流程、状态机、时序、依赖或架构概览。

如果需要精确空间分组、复杂层级、密集注释、非标准形状或明显的视觉主次，不要强迫 Mermaid 承担。

### 精细 SVG

用于概念模型、系统剖面、诊断地图、复杂架构和需要精确布局的解释图。SVG 适合清晰文字、可缩放结构、局部高亮和多层信息，但仍应控制在一个明确问题内。

图中必须有清楚的阅读顺序、分组、标签和图注。关键文字不能小到需要放大才能阅读；不要用装饰性阴影、颜色和图标制造虚假的信息层次。

### 真实数据图表

只有存在真实、可追溯的数值时才使用：

- 随时间变化：折线图或时间轴；
- 类别间数值比较：条形图或点图；
- 分布：直方图、箱线图或密度图；
- 两个数值变量的关系：散点图；
- 多维重复比较：共享尺度的小倍图；
- 矩阵强度：热力图；
- 有真实数量守恒的流动：桑基图或流图；
- 并行阶段、等待与瓶颈：对齐时间轴的泳道图。

图表必须标明来源、单位、时间范围、样本或分母以及必要口径。直接标出读者应关注的值或变化；只有无法直接标注多序列时才加入图例。

没有数据时，改用明确标注的“示意图”，不得编造数值、比例、坐标或趋势来填满画面。

### 交互式 HTML

只在交互本身能产生静态图无法提供的理解时使用，例如：

- 调整参数并观察结果如何变化；
- 在多个状态、方案或时间点之间探索；
- 展开局部细节而保持整体结构；
- 呈现空间运动、动态布局或逐步过程。

首屏在没有任何操作时也必须有用。控件保持最少，一个状态只使用一种控制方式；提供键盘可用性、清晰标签和简短的可访问说明。关键结论不能藏在悬停或点击之后。

如果当前输出环境不支持交互，提供能传达核心结论的静态回退。

### 动画、GIF 或视频

只有时间、顺序、运动或状态转换本身是解释对象时才使用。不要用循环动画、入场特效或装饰性运动吸引注意。

动画应可暂停或逐步查看，首帧有意义，并提供静态图或文字回退。尊重用户减少动态效果的偏好；图表状态切换可以平滑过渡，但不要为首次出现添加无意义动画。

### 生成式位图或插画

适合气氛、隐喻、场景或非精确概念展示，不适合承载需要逐字准确的标签、数字、接口、证据或技术结构。事实图、数据图和技术图优先使用可控的 SVG、图表或 HTML。

## 每张视觉的最小契约

创建前内部回答：

1. 读者要通过这张图看清什么问题？
2. 图要支持哪一个结论或决定？
3. 视觉编码是否对应真实关系，而不是装饰？
4. 数据或外部模型的来源在哪里？
5. 不看图时，读者能否从文字获得关键结论？

交付时：

- 在图前用一句话说明看点，在图后说明它改变了什么理解或判断；
- 给图表、SVG、Canvas 或交互提供简短的标题、描述或文本替代；
- 颜色必须与标签、形状或线型配合，不能成为唯一信息通道；
- 保证文字、标记和控件在常见宽度下不重叠、不裁切；
- 图中表达事实时就地引用；表达作者模型时标为“示意”或“拟议”；
- 一张图只承担一个主要认知任务，多个任务应拆分或回到正文。

## 默认升级路径

按需逐级升级，不跨级追求形式：

`文字或列表 → 表格或 Mermaid → 精细 SVG 或真实数据图表 → 交互式 HTML → 动画`

这不是质量排名。选择标准只有一个：更复杂的媒介是否显著提高理解、判断或操作能力。若答案是否定的，停在更简单的一层。
